API v2.0

API REFERENCE

Control API for air pump (gas lift, 0–100) and water pump (well pressure, 0–100), plus solenoid valve. Sensor ingestion for voltage, current, air pressure, water pressure, temperature, and flow rate.

API Server ● Not tested
1 · Actuator Control
POST/control/air-pump
Set the air pump (gas lift inlet) speed. Controls gas injection into the well. 0 = off, 100 = full speed.
Request Body
valueintegerAir pump level 0–100
{ "value": 75 }
POST/control/water-pump
Set the water pump (well pressure control) speed. 0 = off, 100 = full speed.
Request Body
valueintegerWater pump level 0–100
{ "value": 60 }
POST/control/valve
Open or close the solenoid valve. State is persisted and reflected immediately on the dashboard.
Request Body
statestring"open" or "closed"
{ "state": "open" }
GET/control/state
Read the current commanded state of all actuators. Hardware should call this on boot to sync before running.
GET /control/state
2 · Sensor Data
POST/sensors/reading
Submit a full sensor snapshot from hardware. All 6 fields required. Stored in MongoDB and streamed live to the dashboard.
Request Body
voltagenumberAir pump voltage draw (V) — any value
currentnumberAir pump current draw (A) — any value
air_pressurenumberAir pump output pressure 0–1000
water_pressurenumberWell water pressure near exit 0–1000
temperaturenumberWell temperature near exit 0–1000
flow_ratenumberFlow rate exiting the well 0–1000
{ "voltage": 12.4, "current": 2.85, "air_pressure": 640.0, "water_pressure":780.5, "temperature": 312.0, "flow_rate": 425.5 }
GET/sensors/latest
Returns the most recent sensor snapshot. Returns 404 if no readings exist yet.
GET /sensors/latest
GET/sensors/history
Returns up to the last 100 readings, newest first. Use ?limit=N to request fewer.
limitinteger?Max records — default 100, max 100
GET /sensors/history?limit=20
DELETE/sensors/clear
Permanently deletes all sensor readings. Dashboard resets to zero. Cannot be undone.
DELETE /sensors/clear
3 · Live Stream (SSE)
GET/stream
Server-Sent Events endpoint. The dashboard connects here automatically. Pushes two named events every 2 seconds — sensor and actuator. Use EventSource in the browser or curl to test.
GET /stream Accept: text/event-stream # Test with curl: curl -N http://localhost:3000/stream
GET/health
Returns server status and DB connectivity. Check db field — should be "connected".
GET /health
