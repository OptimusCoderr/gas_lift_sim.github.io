API REFERENCE

Control API for air pump, water pump, and solenoid valve. Sensor ingestion for water flow rate, temperature, and pressure. All endpoints return JSON.

API Server ● Not tested
1 · Actuator Control
POST /control/air-pump
Set the air pump speed via PWM. Value 0 = fully off, 255 = full speed. Hardware responds immediately.
Request Body
pwmintegerPWM duty cycle 0–255
{ "pwm": 128 }
POST /control/water-pump
Set the water pump speed via PWM. Value 0 = fully off, 255 = full speed. Hardware responds immediately.
Request Body
pwmintegerPWM duty cycle 0–255
{ "pwm": 128 }
POST /control/valve
Open or close the solenoid valve. State is applied instantly to hardware and persisted.
Request Body
statestring"open" or "closed"
{ "state": "open" }
GET /control/state
Returns the current commanded state of all actuators — air pump PWM, water pump PWM, and valve state.
GET /control/state
2 · Sensor Data
POST /sensors/reading
Submit a sensor reading from hardware. All three sensor values must be provided. Values are stored in the database and reflected live on the dashboard.
Request Body
flow_ratenumberWater flow rate 0–1000
temperaturenumberTemperature 0–1000
pressurenumberPressure 0–1000
{ "flow_rate": 425.5, "temperature": 312.0, "pressure": 780.3 }
GET /sensors/latest
Returns the single most recent sensor reading from the database.
GET /sensors/latest
GET /sensors/history
Returns up to the last 100 sensor readings ordered newest first. Use the optional limit query parameter to request fewer records.
limitinteger?Max records to return, default 100
GET /sensors/history?limit=20
DELETE /sensors/clear
Permanently deletes all sensor readings from the database. Dashboard values reset to zero. This action cannot be undone.
DELETE /sensors/clear
3 · Health
GET /health
Returns server status and database connectivity. Use for polling from hardware or monitoring tools.
GET /health
